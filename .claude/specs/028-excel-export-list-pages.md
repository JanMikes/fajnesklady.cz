# 028 — Excel export of admin & landlord list pages

**Status:** ready
**Type:** feature (operations / reporting)
**Scope:** medium (~22 files: 1 new dependency, 1 service + 1 value object, 9 single-action controllers, 9 template button additions, 4 repository "all-results" helpers, integration tests)
**Depends on:** none

## Problem

Admins and landlords can read every list in the portal but cannot lift a single row out of it. There is no "Export do Excelu" anywhere — neither for the order ledger, the customer base, the overdue list, the audit/email log, the landlord's revenue invoices, nor a place's storage roster. Operators ask repeatedly for rows they can pivot in Excel (accounting hand-off, debtor follow-ups, inspection prep, support tickets); today the answer is "screenshot or copy/paste row by row".

## Goal

Every list page identified in this spec gains an **Export do Excelu** button next to the existing filter strip. The button:

1. Streams an `.xlsx` download generated by a single shared `ExcelExporter` service.
2. **Mirrors the current URL filter / search params** — no pagination, but every other filter applied on screen is honoured. So `…/orders?filter=individual` → only those rows; `…/users?filter=overdue` → only debtors; etc.
3. **Is owner-scoped identically to the on-screen list.** A landlord's order export is restricted exactly the same way `findByLandlord(User)` already restricts the visible list. PlaceAccess does **not** widen the export — the export shows what the user sees, no more.
4. Filename pattern: `<route-slug>-YYYY-MM-DD.xlsx` (e.g. `objednavky-2026-05-07.xlsx`, `sklady-praha-rozkos-2026-05-07.xlsx`). Diacritics transliterated to ASCII via `transliterator_transliterate('Any-Latin; Latin-ASCII;', $name)` so the filename works in every browser/OS.
5. Czech header row, full diacritics. First row = bold header, columns auto-sized for typical content.

Hard upper bound per export: **50 000 rows**. If the matching set exceeds that, we send the first 50 000 ordered as the on-screen list and add a flash message: `Export omezen na prvních 50 000 řádků — zúžte filtr.` (Hit only by the admin order/email-log/audit-log surfaces in practice; landlord exports are always small.)

### Conventions reminder

- `declare(strict_types=1);` everywhere; `final readonly` for DTOs / value objects; commands/handlers in CQRS dirs (we only add controllers + a service here, no new bus messages).
- Single-action `__invoke` controllers, route at class level, `final` modifier.
- `EntityManager` composition in repositories; never `flush()`; never extend `ServiceEntityRepository`; QueryBuilder over `findBy()`.
- Czech UI text MUST have full diacritics ("objednávky", "po splatnosti", "měsíční platba").
- Money is stored in haléře (`int`); divide by 100 for Kč. Always emit a numeric Kč column (not a string with " Kč" suffix) — easier downstream pivots.
- Turbo is disabled globally; the export button is a plain `<a href>` — no `data-turbo` opt-in needed.

### 2. Value object — `src/Service/Excel/ExcelSheet.php`

```php
namespace App\Service\Excel;

final readonly class ExcelColumn
{
    public function __construct(
        public string $header,
        public ExcelColumnType $type = ExcelColumnType::TEXT,
        // 1.0 = "auto" baseline; the exporter scales per type
        public float $widthHint = 1.0,
    ) {}
}

enum ExcelColumnType: string
{
    case TEXT = 'text';
    case INTEGER = 'integer';
    case DECIMAL = 'decimal';   // 2 decimal places, comma separator handled by Excel locale
    case MONEY_KC = 'money_kc'; // numeric, no Kč suffix in cell
    case DATE = 'date';         // d.m.yyyy
    case DATETIME = 'datetime'; // d.m.yyyy HH:MM
    case BOOLEAN = 'boolean';   // "Ano" / "Ne"
}

final readonly class ExcelSheet
{
    /**
     * @param ExcelColumn[]                                   $columns
     * @param iterable<array<int, scalar|\DateTimeInterface|null>> $rows  // generator allowed
     */
    public function __construct(
        public string $sheetTitle,           // visible in Excel tab strip; max 31 chars
        public string $filename,             // *.xlsx; ASCII-safe, no path separators
        public array $columns,
        public iterable $rows,
        public ?string $truncationFlash = null, // set by exporter if 50 000 cap hit
    ) {}
}
```

Sheet title and filename must be sanitised by callers — the exporter trims sheet title to 31 chars and replaces `[/\\?*\[\]]` characters per Excel rules.

### 3. Service — `src/Service/Excel/ExcelExporter.php`

```php
namespace App\Service\Excel;

use OpenSpout\Common\Entity\Cell;
use OpenSpout\Common\Entity\Row;
use OpenSpout\Common\Entity\Style\CellAlignment;
use OpenSpout\Common\Entity\Style\Style;
use OpenSpout\Writer\Common\Entity\Sheet as OpenSpoutSheet;
use OpenSpout\Writer\XLSX\Options;
use OpenSpout\Writer\XLSX\Writer;
use Symfony\Component\HttpFoundation\StreamedResponse;

final readonly class ExcelExporter
{
    public const int MAX_ROWS = 50_000;

    public function stream(ExcelSheet $sheet): StreamedResponse
    {
        $response = new StreamedResponse(function () use ($sheet): void {
            $options = new Options();
            $options->DEFAULT_COLUMN_WIDTH = 18;

            $writer = new Writer($options);
            $writer->openToFile('php://output');
            $writer->getCurrentSheet()->setName($this->safeSheetTitle($sheet->sheetTitle));

            // Bold header
            $writer->addRow(Row::fromValues(
                array_map(static fn (ExcelColumn $c) => $c->header, $sheet->columns),
                (new Style())->setFontBold(),
            ));

            // Body — type-aware cell mapping; never let PHP's locale dictate decimals
            $written = 0;
            foreach ($sheet->rows as $rawRow) {
                if ($written >= self::MAX_ROWS) {
                    break;
                }
                $writer->addRow(Row::fromValues(
                    $this->mapValues($rawRow, $sheet->columns),
                ));
                $written++;
            }

            $writer->close();
        });

        $response->headers->set(
            'Content-Type',
            'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
        );
        $response->headers->set(
            'Content-Disposition',
            sprintf('attachment; filename="%s"', $this->safeFilename($sheet->filename)),
        );
        $response->headers->set('Cache-Control', 'no-store, no-cache, must-revalidate');
        $response->headers->set('Pragma', 'no-cache');

        return $response;
    }

    /** @param array<int, scalar|\DateTimeInterface|null> $row @param ExcelColumn[] $columns @return array<int, scalar|\DateTimeInterface|null> */
    private function mapValues(array $row, array $columns): array { /* per ExcelColumnType */ }

    private function safeSheetTitle(string $title): string { /* 31-char cap, strip [/\?*[]] */ }

    private function safeFilename(string $filename): string { /* ASCII-safe, transliterate */ }
}
```

Implementation notes for the dev:

- **Date / datetime cells**: openspout v4 supports `\DateTimeInterface` directly via `Cell::fromValue($dt, $dateStyle)`. Apply a `numberFormat: 'dd.mm.yyyy'` (or `'dd.mm.yyyy hh:mm'`) style once, reuse the style instance — do NOT create one per cell.
- **Money cells**: integer haléře get divided by 100 and emitted as a float; no Kč suffix in the cell itself (header reads `… (Kč)`). Two-decimal format `'#,##0.00'` so it reads correctly under any user locale.
- **Boolean cells**: `bool` → `'Ano' | 'Ne'` (text). Do not emit `1` / `0` — admin/landlord users are non-technical.
- **NULL cells**: emit empty string (openspout's `null` is fine; just be consistent).
- **Truncation flash**: if `$written` hit the cap, set a `Last-Modified` no-op header AND stash a flash message on the session via `RequestStack` injected in the exporter — actually no, the controller can't add a flash after `StreamedResponse` starts. **Cleaner approach**: each export controller, before invoking the exporter, calls a count helper on the same repository; if `count >= 50 000`, the controller adds the flash before redirecting back to the list with a `?export=truncated` notice instead of streaming. **But** that doubles every query. Final decision: **skip the flash** — the cap is documented behaviour and the cell A1 of the sheet contains `Sklady (prvních 50 000)` when truncated, set by the controller building the title. Keep the exporter pure.

### 4. Generic export controller pattern — single-action, owner-scoped

Every export controller MUST:

1. Carry the **same `#[IsGranted('ROLE_ADMIN')]` / `#[IsGranted('ROLE_LANDLORD')]`** as the source list.
2. For landlord-scoped exports that touch a Place subject: call `$this->denyAccessUnlessGranted(PlaceVoter::VIEW, $place)` with the same `Uuid::fromString($placeId)` lookup.
3. For per-owner data: pull `$user = $this->getUser()` and pass it to the **same repository method** the list uses (e.g. `findByLandlord($user)`, `findFiltered($user, $place, $type)`).
4. Read the same `Request` query params the list reads. **No** new filter axes; export is dumb pass-through.
5. Build `ExcelSheet` with the columns specified per page below; rows are produced by a `static function (...): \Generator` so memory stays bounded for the admin lists.
6. Return `$this->excelExporter->stream($sheet)`.

A common bottom-of-controller helper (`private function nowSlug(): string` returning `Y-m-d`) is fine to inline — do not add a base class. Symfony controllers don't gain from inheritance here.

/**
 * Streamed iteration for export. Honours the same admin filter as findAdminFiltered.

 * @return iterable<Order>

public function streamAdminFiltered(\DateTimeImmutable $now, ?string $filter): iterable
{
    $qb = $this->buildAdminFilteredQueryBuilder($now, $filter)
        ->orderBy('o.createdAt', 'DESC');

    foreach ($qb->getQuery()->toIterable() as $order) {
        yield $order;
        $this->entityManager->clear();   // bound peak memory; safe — we don't mutate

}

## Out of scope

- **CSV / ODS export.** XLSX-only. Adding CSV is two lines (separate controller + `OpenSpout\Writer\CSV\Writer`) but no one asked; defer.
- **Background jobs / e-mailed exports.** Synchronous streamed download is enough at our scale. Beyond 50 000 rows, the user is told to narrow the filter — we don't introduce Messenger queues here.
- **Saved column profiles / user-selectable columns.** Each export's column set is fixed in code. UI for "pick columns" adds churn no one asked for.
- **Branded XLSX template (logo, footer).** A spreadsheet for accountants doesn't need branding; first row bold + frozen is enough.
- **Customer-facing export** (a tenant exporting their own orders). Out — this spec is operations-only.
- **Calendar export.** `/portal/calendar` is visual; an iCal feed would be a different feature.
- **Storage-type list export.** Pure CRUD config — low value; a dev who needs it dumps from `psql` faster.
- **Place list (`/portal/places`) export for landlords.** It's a browse-anyone-else's-place view; the data is the same as the public detail and not useful in tabular form. Admin places list IS exported (it carries owner + counts).
- **Pivot tables / charts inside the XLSX.** Output is flat rows; downstream user pivots in Excel.
- **Concurrent-export rate limiting / abuse protection.** A streamed XLSX is cheap; admins/landlords are trusted users. Skip.
